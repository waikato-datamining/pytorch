# ast

[ast](https://github.com/YuanGongND/ast), uses PyTorch 1.11.0 for GPU.


## Quick start

### Directories

Create the following directories where you want to start the Docker image from:

```
mkdir cache
```

### Inhouse registry

* Pull and run image (adjust volume mappings `-v`):

  ```bash
  docker run --rm --gpus=all --shm-size 8G --net=host \
    -v `pwd`:/workspace \
    -v `pwd`/cache:/cache \
    -it harbor.cms.waikato.ac.nz/public/pytorch/pytorch-ast:2023-05-22_cuda12.3
  ```

### Docker hub

* Pull and run image (adjust volume mappings `-v`):

  ```bash
  docker run --rm --gpus=all --shm-size 8G --net=host \
    -v `pwd`:/workspace \
    -v `pwd`/cache:/cache \
    -it waikatodatamining/pytorch-ast:2023-05-22_cuda12.3
  ```

### Build local image

* Build the image from Docker file (from within /path_to/ast/2023-05-22_cuda12.3)

  ```bash
  docker build -t ast .
  ```
  
* Run the container

  ```bash
  docker run --rm --gpus=all --shm-size 8G --net=host \
    -v `pwd`:/workspace \
    -v `pwd`/cache:/cache \
    -it ast
  ```
  `/local/dir:/container/dir` maps a local disk directory into a directory inside the container


## Publish images

### Build

```bash
docker build -t ast:2023-05-22_cuda12.3 .
```

### Inhouse registry  

* Tag

  ```bash
  docker tag \
    ast:2023-05-22_cuda12.3 \
    harbor.cms.waikato.ac.nz/public/pytorch/pytorch-ast:2023-05-22_cuda12.3
  ```
  
* Push

  ```bash
  docker push harbor.cms.waikato.ac.nz/public/pytorch/pytorch-ast:2023-05-22_cuda12.3
  ```
  If error "no basic auth credentials" occurs, then run (enter username/password when prompted):
  
  ```bash
  docker login harbor.cms.waikato.ac.nz
  ```

### Docker hub  

* Tag

  ```bash
  docker tag \
    ast:2023-05-22_cuda12.3 \
    waikatodatamining/pytorch-ast:2023-05-22_cuda12.3

  ```
  If error "no basic auth credentials" occurs, then run (enter username/password when prompted):

### Requirements

```bash
docker run --rm --pull=always \
  -it harbor.cms.waikato.ac.nz/public/pytorch/pytorch-ast:2023-05-22_cuda12.3 \
  pip freeze > requirements.txt
```


## Permissions

When running the docker container as regular use, you will want to set the correct
user and group on the files generated by the container (aka the user:group launching
the container):

```bash
docker run -u $(id -u):$(id -g) -e USER=$USER ...
```


## Scripts

The following additional scripts are available:

* `ast_predict_poll` - batch-processing of audio files via file-polling
* `ast_predict_redis` - making predictions via Redis backend


### ast_predict_redis
 
You need to start the docker container with the `--net=host` option if you are using the host's Redis server.
