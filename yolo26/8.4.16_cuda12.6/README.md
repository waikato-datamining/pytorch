# Yolo26

[Yolo26](https://docs.ultralytics.com/models/yolo26/), 
uses PyTorch 2.x and Ultralytics 8.4.16

## Quick start

### Directories

Create the following directories where you want to start the Docker image from:

```
mkdir cache
mkdir config
```

### Inhouse registry

* Pull and run image (adjust volume mappings `-v`):

  ```bash
  docker run --rm --gpus=all --shm-size 8G --net=host \
    -v `pwd`:/workspace \
    -v `pwd`/cache:/cache \
    -v `pwd`/config:/config \
    -it harbor.cms.waikato.ac.nz/public/pytorch/yolo26:8.4.16_cuda12.6
  ```

### Docker hub

* Pull and run image (adjust volume mappings `-v`):

  ```bash
  docker run --rm --gpus=all --shm-size 8G --net=host \
    -v `pwd`:/workspace \
    -v `pwd`/cache:/cache \
    -v `pwd`/config:/config \
    -it waikatodatamining/pytorch-yolo26:8.4.16_cuda12.6
  ```

### Build local image

* Build the image from Docker file (from within /path_to/yolo26/8.4.16_cuda12.6)

  ```bash
  docker build -t yolo26 .
  ```
  
* Run the container

  ```bash
  docker run --rm --gpus=all --shm-size 8G --net=host \
    -v `pwd`:/workspace \
    -v `pwd`/cache:/cache \
    -v `pwd`/config:/config \
    -it yolo26
  ```


## Publish images

### Build

```bash
docker build -t yolo26:8.4.16_cuda12.6 .
```

### Inhouse registry  

* Tag

  ```bash
  docker tag \
    yolo26:8.4.16_cuda12.6 \
    harbor.cms.waikato.ac.nz/public/pytorch/yolo26:8.4.16_cuda12.6
  ```
  
* Push

  ```bash
  docker push harbor.cms.waikato.ac.nz/public/pytorch/yolo26:8.4.16_cuda12.6
  ```
  If error "no basic auth credentials" occurs, then run (enter username/password when prompted):
  
  ```bash
  docker login harbor.cms.waikato.ac.nz:443
  ```

### Docker hub  

* Tag

  ```bash
  docker tag \
    yolo26:8.4.16_cuda12.6 \
    waikatodatamining/pytorch-yolo26:8.4.16_cuda12.6
  ```
  
* Push

  ```bash
  docker push waikatodatamining/pytorch-yolo26:8.4.16_cuda12.6
  ```
  If error "no basic auth credentials" occurs, then run (enter username/password when prompted):
  
  ```bash
  docker login
  ``` 


### Requirements

```bash
docker run --rm --pull=always \
  -it harbor.cms.waikato.ac.nz/public/pytorch/yolo26:8.4.16_cuda12.6 \
  pip freeze > requirements.txt
```


## Permissions

When running the docker container as regular use, you will want to set the correct
user and group on the files generated by the container (aka the user:group launching
the container):

```bash
docker run -u $(id -u):$(id -g) -e USER=$USER ...
```


## Scripts

The following additional scripts are available:

* `yolo26_train` - the yolo26 script for training models
* `yolo26_export` - the yolo26 script for exporting models (eg to ONNX)
* `yolo26_predict_poll` - batch-processing of images via file-polling
* `yolo26_predict_redis` - making predictions via Redis backend


### yolo26_predict_redis
 
You need to start the docker container with the `--net=host` option if you are using the host's Redis server.
